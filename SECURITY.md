## 报告安全漏洞

我们非常重视 ZQuant 项目的安全性。如果您发现了安全漏洞，请按照以下流程报告：

### 如何报告

**请勿在公开的 GitHub Issues 中报告安全漏洞！**

请通过以下方式私下报告：

1. **发送邮件**: kevin@vip.qq.com
   - 主题请包含 "[SECURITY]" 前缀
   - 详细描述漏洞情况
   - 如果可能，提供复现步骤

2. **使用 GitHub Security Advisories** (推荐)
   - 访问: https://github.com/zquant/zquant/security/advisories
   - 点击 "Report a vulnerability"
   - 填写漏洞详情

### 报告内容

请在报告中包含以下信息：

- 漏洞类型和影响范围
- 复现步骤（代码、配置等）
- 潜在的影响和风险
- 建议的修复方案（如果有）

### 响应时间

我们承诺：

- **初步响应**: 在收到报告后 48 小时内确认收到
- **状态更新**: 每周至少更新一次处理状态
- **修复时间**: 根据漏洞严重程度，在合理时间内发布修复

### 漏洞严重程度

我们使用以下标准评估漏洞严重程度：

- **严重 (Critical)**: 可能导致数据泄露、系统完全被控制等
- **高危 (High)**: 可能导致部分数据泄露、权限提升等
- **中危 (Medium)**: 可能导致信息泄露、拒绝服务等
- **低危 (Low)**: 影响较小，如信息泄露等

### 披露政策

- 在漏洞修复并发布补丁后，我们会公开披露漏洞信息
- 我们会给报告者适当的致谢（如果报告者同意）
- 我们不会在修复前公开披露漏洞细节

### 奖励

目前项目暂不提供漏洞奖励计划，但我们会在发布说明中致谢安全研究人员。

### 安全最佳实践

使用 ZQuant 时，请遵循以下安全最佳实践：

1. **及时更新**: 保持项目版本为最新
2. **环境变量**: 妥善保管敏感配置（数据库密码、API密钥等）
3. **网络安全**: 在生产环境中使用 HTTPS
4. **访问控制**: 合理配置用户权限
5. **日志监控**: 定期检查日志，发现异常活动
